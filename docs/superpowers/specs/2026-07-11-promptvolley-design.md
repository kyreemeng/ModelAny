# ModelAny P0 + P1 设计规范

日期：2026-07-11  
状态：已批准  
产品名称：ModelAny

## 1. 范围与目标

首版实现 PRD 中全部 P0 与 P1 功能，交付可构建、可测试、可通过 Chrome“加载已解压的扩展程序”运行的 Manifest V3 插件。

包含：

- 5000 Unicode 字符以内的多行提问输入、计数、纯文本粘贴和草稿恢复
- 豆包、通义千问、DeepSeek、Kimi、智谱 GLM 五模型选择
- 一键打开后台标签页、自动填充，并按设置决定是否提交
- 全选/清空、Cmd/Ctrl+Enter、自动提交、标签分组
- 最近 20 条历史、单条删除和全部清空
- 选中文字后的右键菜单入口
- 设置、诊断、最近 50 条发送日志及 JSON 导出
- 部分成功、超时、未登录、页面结构变化等异常反馈

不包含 P2：模型预设、自定义模型、暗色主题和 Firefox 适配。

PRD 中残留的 “PromptCaster” 文案统一修正为 “ModelAny”。模型图标首版使用品牌色字母图形，不使用可能存在版权风险的官方 Logo。

## 2. 技术方案

采用原生 TypeScript + Vite，多入口构建，无 UI 运行时框架。

### 2.1 运行单元

- `popup`：主输入、模型选择、历史抽屉、发送状态和失败详情
- `options`：通用设置、模型管理、数据管理、诊断、日志与关于
- `service worker`：标签页创建、加载等待、消息调度、结果汇总、右键菜单和标签分组
- 五个 content-script 入口：每个模型一个入口，复用共享填充引擎
- `shared`：模型定义、消息类型、存储仓库、验证器和错误映射

### 2.2 Manifest 与权限

使用 Manifest V3。权限仅包含：

- `storage`
- `tabs`
- `tabGroups`
- `scripting`
- `contextMenus`
- `alarms`

Host 权限仅覆盖：

- `https://www.doubao.com/*`
- `https://tongyi.aliyun.com/*`
- `https://chat.deepseek.com/*`
- `https://kimi.moonshot.cn/*`
- `https://chatglm.cn/*`

不加载远程脚本，不使用 `eval`，不上传任何数据。

### 2.3 Service Worker 可靠性

不依赖全局变量保存任务。每次发送创建持久化任务记录，其中包含任务 ID、文本摘要、目标模型、自动提交设置、各模型状态、标签页 ID、开始时间和截止时间。

每次状态变化立即写入 `chrome.storage.local`。Worker 被系统回收后，在启动、消息到达或 alarm 唤醒时恢复未完成任务，并根据标签页和截止时间继续或结束任务。`chrome.alarms` 负责唤醒超时清理，不被当作持续保活机制。

## 3. 数据模型

### 3.1 用户状态

- `draftText: string`
- `modelEnabled: Record<ModelId, boolean>`
- `settings.autoSubmit: boolean`
- `settings.groupTabs: boolean`
- `settings.confirmManyTabs: boolean`
- `settings.localMetrics: boolean`

默认值：五模型全开、自动提交开启、标签分组开启、五标签确认开启、本地指标开启。

### 3.2 历史

最多 20 条，按时间倒序：

- `id`
- `text`
- `createdAt`
- `modelIds`
- `autoSubmit`

相同文本再次发送时新增记录；超过上限删除最早记录。

### 3.3 日志

最多 50 条：

- `id`
- `taskId`
- `modelId`
- `startedAt`
- `durationMs`
- `result`
- `errorCode`
- `errorMessage`

日志不保存完整提问文本，只关联任务并记录必要诊断信息。导出文件包含版本、导出时间、浏览器信息、设置摘要和日志。

## 4. 发送数据流

1. Popup 对文本和模型选择做同步校验。
2. 选择五个模型且确认设置开启时，显示确认对话框。
3. Popup 保存草稿和历史，并向 Worker 发送不可变的任务快照。
4. Worker 按 PRD 顺序创建非活动标签页，各模型任务并行等待。
5. 标签页加载完成后，Worker 向对应 content script 发送填充命令。
6. Content script 检查登录页，轮询输入框，使用原生 setter 或 contenteditable 插入策略写入文本，并触发 `input`、`change` 事件。
7. 自动提交开启时，等待界面状态更新并点击可用的发送按钮；关闭时只填充。
8. 每个模型独立返回结构化结果；单模型总超时 15 秒，输入框查找超时 10 秒。
9. Worker 持久化每个结果并向 Popup 广播进度。
10. 全部结束后创建标签组、写入日志，并返回成功、部分成功或失败摘要。

Popup 被关闭不会取消发送任务。重新打开 Popup 时可从存储恢复当前任务进度。

## 5. Content Script 设计

每个模型适配器包含：

- 模型 ID 和允许的主机名
- 输入框主选择器和备用选择器
- 提交按钮主选择器和备用选择器
- 登录 URL 特征和登录页面元素特征
- 页面额外就绪时间
- 可选的模型专属填充或提交钩子

共享引擎负责：

- 验证消息来源和文本长度
- 在 10 秒内按 500ms 间隔轮询可见、可编辑输入框
- 支持 `textarea`、`input` 和 `[contenteditable=true]`
- 触发框架可识别的输入事件
- 等待提交按钮变为可用
- 返回 `FILLED`、`SUBMITTED`、`NOT_LOGGED_IN`、`INPUT_NOT_FOUND`、`SUBMIT_NOT_FOUND` 或 `UNEXPECTED_ERROR`

选择器优先使用 `data-testid`、`aria-label`、`role`、`placeholder` 等语义属性，类名仅作为末级备用。模型适配器相互隔离，网页改版时只修改对应配置。

## 6. 交互与视觉

选择 B“轻盈 AI 感”：

- 400px Popup，白色半透明表面，淡蓝至淡紫背景
- 主色采用 `#5B70FF` 到 `#8A67ED` 的轻微渐变
- 12–16px 圆角，柔和阴影，8px 基础间距网格
- 系统字体，正文 14px，保证 WCAG AA 对比度
- 模型使用品牌色圆形字母图标
- 动效控制在 120–180ms，并尊重 `prefers-reduced-motion`

### 6.1 Popup

顺序为：

1. 品牌栏与设置入口
2. 自动扩展输入框和字符计数
3. 模型阵容标题、全选和清空
4. 2 列模型卡片；开启时高亮，关闭时弱化
5. 自动提交与历史入口
6. 全宽渐变发送按钮和快捷键提示
7. 版本与隐私说明

输入框最小 120px，最大 260px，Popup 总高最大 600px。历史以内部抽屉覆盖模型区域，避免打开新页面。Toast 固定在 Popup 顶部，不遮挡发送按钮。

### 6.2 状态

- 空文本：按钮禁用，文案“请输入问题”
- 无模型：按钮禁用，文案“请选择至少一个模型”
- 就绪：文案“发送到 N 个模型”
- 发送中：显示进度 “已完成 M/N”，禁止重复发送
- 全成功：绿色 Toast，提供“查看结果”
- 部分成功：橙色 Toast 和可展开失败详情
- 全失败：红色 Toast 和逐模型处理建议

### 6.3 设置页

使用分区卡片：通用设置、模型管理、数据管理、诊断、关于。诊断逐模型显示空闲、检测中、正常或异常。危险操作使用二次确认。

## 7. 右键菜单

安装时创建“使用 ModelAny 提问”菜单，仅在有选中文本时出现。点击后将选中文字裁剪到 5000 字符并保存为草稿，然后调用 `chrome.action.openPopup()`。

若当前 Chrome 版本或上下文不允许自动打开 Popup，则设置扩展徽标提示，并发送系统通知式状态到扩展图标；用户点击图标后可看到已填入的草稿。该降级不自动发送。

## 8. 错误处理

- 输入仅空白：不发送
- 超过 5000 字符：输入和粘贴均按 Unicode code point 截断并提示
- 标签页创建失败：仅该模型失败
- 标签页被关闭：记录 `TAB_CLOSED`
- 页面 15 秒未完成：记录 `PAGE_TIMEOUT`
- 登录页：记录 `NOT_LOGGED_IN` 并提供打开链接
- 输入框缺失：记录 `INPUT_NOT_FOUND`，提示网页可能更新
- 自动提交按钮缺失：保留已填文本，记录 `SUBMIT_NOT_FOUND`
- Storage 不可用：当前 Popup 会话继续工作，但明确提示状态不会保存
- Popup 关闭：后台任务继续，不依赖 Popup 连接

所有外部消息均校验类型、模型 ID、文本长度和发送方标签页主机名。

## 9. 诊断

诊断为显式用户操作。每个模型打开后台标签页，先在可编辑输入框中填入固定文本 `ModelAny`（不提交），再检测：

- 页面是否可加载
- 是否处于登录页
- 是否能找到可编辑输入框
- 自动提交按钮是否可找到

诊断不会点击提交。完成后可保留标签页供用户登录或检查；设置页提供关闭本次诊断标签页操作。
